# Web3Proof V1.0 —— 基于区块链的学生成果存证与核验平台

# 一、项目概述

## 1.1 项目背景

随着高校学生实践活动、学科竞赛、课程项目、实习经历和技能证书数量不断增加，学生在求职、评奖评优、升学推荐和成果归档过程中需要频繁提交各类证明材料。然而传统材料核验主要依赖纸质证书、截图、简历描述或人工联系核实，存在材料易伪造、核验流程繁琐、证明来源分散、第三方难以快速确认真实性等问题。

区块链具有不可篡改、可追溯、公开可验证等特点，适合用于数字材料的可信存证与核验。本系统设计并实现一个基于区块链的学生成果存证与核验平台，通过 DID 身份标识、IPFS 文件存储、链上哈希存证和 SBT 不可转让认证凭证，为学生成果材料提供可信记录和第三方核验能力。

系统面向学生、管理员和核验方三类用户。学生可以连接钱包登录，维护个人资料，上传项目成果、竞赛证书、实习证明、课程作品等材料，并将材料摘要写入区块链完成存证。管理员负责审核学生提交的成果认证申请，并在审核通过后发放链上认证凭证。核验方可以根据学生钱包地址或核验链接查看学生的可信成果档案，验证材料哈希、认证记录和链上交易信息。

本系统旨在解决学生成果材料真实性难以确认、人工核验成本高、证明记录分散的问题，形成一个集成果上传、链上存证、审核认证、证书发放、可信核验于一体的学生成果可信认证平台。

---

# 1.2 项目目标

本项目旨在构建一个面向高校学生成果管理与核验的可信平台，实现以下目标：

* 实现学生钱包登录与 DID 身份登记
* 实现项目成果、竞赛证书、实习证明等材料的 IPFS 存储与链上哈希存证
* 建立管理员审核机制，对学生提交的成果材料进行认证
* 使用 SBT 生成不可转让的链上认证凭证
* 为企业、教师或第三方机构提供学生成果真实性核验服务
* 生成学生可信成果档案，降低材料造假和人工核验成本

# 1.3 项目定位

Web3Proof V1.0 的定位为：

> 一个基于区块链的学生成果存证与第三方核验平台。

系统主要服务于以下用户群体：

* 创建链上身份
* 上传成果材料
* 完成链上存证

* 验证成果材料真实性
* 查询链上交易与认证凭证
* 生成材料核验报告

# 二、系统总体架构

系统采用前后端分离架构，链下保存业务索引和展示数据，链上保存关键哈希、凭证和可验证记录。

```text
学生/管理员/核验方
 │
 ▼
React 前端
 │
 ▼
Go 后端（Gin）
 │
 ├── SQLite/MySQL
 ├── Redis
 │
 ▼
IPFS / Pinata
 │
 ▼
Ethereum Sepolia
 │
 ▼
Smart Contracts
```

## 2.1 技术栈

* 前端：React、TypeScript、Vite、Ant Design、Ethers.js
* 后端：Go、Gin、Gorm、JWT
* 区块链：Solidity、Hardhat、OpenZeppelin、Ethereum Sepolia
* 存储：SQLite/MySQL、Redis、IPFS/Pinata
* 部署：Docker、Docker Compose、Nginx

# 三、核心功能模块

## 3.1 用户身份管理模块

学生、管理员和核验方通过 MetaMask 钱包登录系统。学生可登记 DID，形成唯一链上身份标识。

主要功能：

* 钱包登录
* JWT 会话认证
* DID 创建与链上登记
* 个人资料维护
* 角色权限控制

## 3.2 成果存证模块

学生上传课程项目、竞赛证书、实习证明、技能证书、科研成果等材料。系统将材料元数据上传 IPFS，生成 CID 和内容哈希，再由学生钱包签名写入区块链。

存证流程：

```text
填写成果信息
   ↓
上传 IPFS
   ↓
生成 CID 和内容哈希
   ↓
MetaMask 确认链上登记
   ↓
完成可信存证
```

## 3.3 成果认证模块

学生选择成果类别并提交证明材料，管理员根据材料真实性和完整性进行审核。审核通过后，系统进入 SBT 凭证发放流程。

默认成果类别：

* 课程项目
* 学科竞赛
* 实习证明
* 技能证书
* 科研成果

## 3.4 SBT 认证凭证模块

系统采用 Soulbound Token 作为成果认证凭证。SBT 不可转让、不可交易，适合表达“某学生已通过某项成果认证”的链上记录。

特点：

* 不可转让
* 不可交易
* 链上可验证
* 与学生钱包地址绑定

## 3.5 可信评分模块

系统根据学生成果存证数量、认证凭证数量和资料完整度生成可信评分，辅助核验方快速判断档案完整性。

| 维度 | 权重 |
| ---- | --- |
| 成果存证 | 50% |
| 认证凭证 | 30% |
| 资料完整度 | 20% |

## 3.6 可信成果档案模块

系统聚合学生 DID、成果存证记录、SBT 认证凭证和可信评分，生成可公开访问的可信成果档案。

展示内容：

* DID 与钱包地址
* 学生基础资料
* 成果存证记录
* SBT 认证凭证
* 可信评分与评级

## 3.7 成果核验模块

核验方可输入学生钱包地址，查看学生可信成果档案并生成核验报告。报告包含身份状态、成果数量、认证凭证、链上记录和综合评级。

核验报告示例：

```text
身份认证：已完成
成果数量：5
认证凭证：3
可信评分：820
综合评级：A
```

# 四、智能合约设计

现有合约保持不重命名，避免重新部署和迁移；业务展示层改为学生成果语义。

## DIDRegistry.sol

负责学生 DID 身份登记与资料更新。

核心功能：

```solidity
registerDID()
updateProfile()
getProfile()
```

## ProjectRegistry.sol

负责成果材料哈希存证。内部仍使用 `Project` 结构，但业务含义为学生成果材料。

核心功能：

```solidity
addProject()
getProject()
verifyProject()
```

## SkillSBT.sol

负责成果认证凭证发放。内部仍使用 `SkillSBT` 和 `skillType`，展示层解释为成果类别与认证凭证。

核心功能：

```solidity
mint()
burn()
verify()
```

## Reputation.sol

负责可信评分上链同步。

核心功能：

```solidity
updateScore()
getScore()
```

# 五、系统创新点

## 创新点一

基于 DID 的学生链上身份体系，实现学生成果档案的身份可信绑定。

## 创新点二

采用 IPFS + 区块链实现成果材料哈希存证，兼顾材料存储与链上可验证。

## 创新点三

利用 Soulbound Token 构建不可转让的成果认证凭证，避免证书转让和冒用。

## 创新点四

自动生成可信成果档案，为企业、教师和第三方机构提供低成本核验入口。
